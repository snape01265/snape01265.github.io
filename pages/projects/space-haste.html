Experience and Feelings

This project was created for the Capstone Project of KRAFTON Jungle bootcamp.

From a programming perspective, building this game gave me insights into how a client should be structured for a networked environment.

As the project lead, I learned about the importance of collaboration and communication between team members, scheduling and task delegation, and how to effectively convey a game design vision to the other members. This project taught me what it takes to lead a team.


Gameplay Features

  • Spaceship controls and movement
  • Player camera
  • UI input handling, animations, and HUD
  • Leaderboard, position tracking and real-time ranking display
  • Laser shooting and health system
  • Game flow from lobby to in-game
  • Shader and Visual Effects
  • Scene lighting (point & global) and volume-based Post-Processing

    Memorable Things from Development

    Natural Movement & Effects

    Applied vector math, calculus, and easing functions to achieve natural-feeling object movement and effects throughout the game. Getting the spaceship to feel responsive yet weighty in zero-gravity required lots of tuning and testing. I think I tuned the movement until the very last moment, right before presentation.

    Understanding Multiplayer Games

    Explored the fundamentals of networked games. Mainly, how client and server communicate state, the struggles of latency and synchronization, and the trade-offs between authoritative and client-predicted models. Even in a small project like this, designing systems that could function correctly in a multiplayer context taught me how games handle real-time data exchange.
